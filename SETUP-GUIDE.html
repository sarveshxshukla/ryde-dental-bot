E

Optional polish (when you're ready)

Two upgrades — neither is required to launch

1 · Remove the "slow first message"

Smily already wakes the server early and captures details during any wait, so most visitors won't notice. If the clinic gets late-night traffic and you want it instant 24/7, in Render open the service → Settings → change the instance from Free to Starter (about US$7/month ≈ A$11). That removes sleeping entirely.

2 · Email every chat + keep a permanent record

Smily can email you each booking the moment it happens and a transcript of every chat, and save everything to a Google Sheet so nothing's ever lost. The kit includes google-apps-script.gs with ~5-minute setup notes inside — paste it into a new Google Sheet, deploy it as a Web app, and add its URL to Render as NOTIFY_WEBHOOK_URL. (Full steps are under "EMAIL-SETUP" in the README.) Prefer email-only? A free Web3Forms key in WEB3FORMS_KEY also works.

⚠️Recommended before you rely on it: set up the email + Sheet logging above, so every booking has a permanent home — on the free tier the in-app list can reset when the server sleeps.

If something's off

Bubble says "Setup needed" → the GEMINI_API_KEY isn't set in Render. Re-check it under the service's Environment settings.
Inbox says "Wrong password" → the word you typed doesn't match the ADMIN_TOKEN in Render's settings.
No bubble on the website → make sure the <script> line uses your real Render address and ends in /widget.js, and that the WPCode snippet is set to Active.
First message very slow → the free server was asleep; it wakes in ~30–60s, then it's instant. Stage E removes this.
A "high demand / 503" reply → Google's model was briefly busy. Smily now retries automatically — usually it just takes a beat longer.
